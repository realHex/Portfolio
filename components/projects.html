Event Ticket Booking System

A real-time multi-threaded ticket booking simulation that allows multiple vendors to add tickets, and multiple users to buy tickets. The system is designed to handle multiple users and vendors concurrently.




Features

  • Provides input fields, configuration options, and control buttons.
  • Shows ticket status, transaction logs, and real-time updates while logging transactions and configurations.
  • Polls the backend to fetch and display updates, implementing multi-threaded vendor and customer operations.
  • Thread safety is ensured using ReentrantLock for managing ticket additions and purchases.
  • REST APIs enable frontend-backend communication, supporting real-time ticket sales updates with charts.
  • System stores transaction data in a database and allows adding vendors/customers at runtime, including priority ticket access for VIP customers.

Technologies used: Java, Springboot, Angular, MongoDB





Quized - A quiz application

A quiz application that allows users to upload their own question sets and take quizzes. The user can upload csv files containing the question set. The application will parse the csv file and store the questions in the database. The user can then attempt that quiz.




Features

  • Allows user to upload CSV files containing questions
  • Saves the quizzes to the database
  • Allows attempting those quizzes and stores the score in History
  • Allows deleting quizzes the user uploaded themself
  • Allows other users to attempt another user's quizzes

Technologies used: React, Next.js, Supabase
